# 참고 자료 — 데이터 수집·표준화 (은아 담당분)

코드에 실제로 적용된 기법의 출처 정리. 학술 논문보다는 표준 스펙/오픈소스 라이브러리 레퍼런스 위주.

| 기법 | 출처 | 코드 위치 | 용도 |
|---|---|---|---|
| Readability 알고리즘 | Mozilla `Readability.js` (Arc90 Readability 계승, 오픈소스) | `chrome-extension/lib/Readability.js` | 브라우징한 뉴스/기사 페이지에서 본문만 자동 추출 |
| OAuth 2.0 Authorization Framework | RFC 6749 (IETF) | `backend/routers/auth.py`, `collectors/spotify_collector.py`, `calendar_collector.py` | Spotify/Google 인증·토큰 갱신 흐름 |
| EXIF (Exchangeable Image File Format) | CIPA DC-008 표준 규격 | `backend/collectors/photo_processor.py` | 사진 촬영시각·GPS·카메라 모델 메타데이터 추출 |
| Google Cloud Vision API — TEXT_DETECTION | Google Cloud 공식 문서 | `backend/collectors/photo_processor.py` | 스크린샷 OCR |

## 참고자료 목록에서 빼는 게 나은 것

`normalize.py`(6개 소스 표준화 스키마 통합)와 PII 마스킹(`pii_mask.py`)은 특정 논문을 따른 게 아니라 정규식 기반의 일반적인 엔지니어링 구현. 억지로 논문을 갖다붙이면 심사에서 "왜 이 논문?"이라고 물었을 때 답이 궁색해질 수 있음 — 굳이 넣는다면 "표준 ETL 패턴 적용" 정도로 낮은 수준으로만 언급.
